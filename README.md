# OpenKoma

OpenKoma 是一个纯本地的漫画分镜编辑器，专门服务于“把已有图片快速排成漫画页面”这件事。

它不再内置任何后端服务，也不依赖在线生图、去背景或超分接口；当前定位非常明确：

- 导入本地图片
- 调整分镜形状与布局
- 进行非破坏裁剪
- 添加文字气泡
- 保存、加载并导出页面

如果你已经有角色图、背景图、对白素材，OpenKoma 的目标就是让你更快地完成漫画页面编排，而不是把软件做成一个复杂的云端工作台。

## 项目特点

- 真正的斜切分镜：支持矩形、平行四边形、梯形，不只是旋转矩形。
- 非破坏裁剪：原图始终保留，显示效果只由裁剪参数驱动。
- 裁剪形状跟随分镜：手动裁剪框会对齐当前分镜的真实几何形状。
- 透明图片支持：本地 PNG / WebP / GIF / SVG / AVIF 的透明通道可保留显示。
- 多页面工作流：支持新增页面、页面排序，并按页面顺序导出 PDF。
- 本地优先存储：支持目录模式保存，也支持单文件 `.openkoma.json` 导入导出。
- 增量撤销重做：编辑历史基于 JSON Patch 记录，适合高频迭代调整。
- 文字气泡系统：支持矩形、圆角、圆形气泡，支持横排/竖排文字与边框样式调整。

## 当前定位

OpenKoma 当前是一个“轻量但几何能力很强”的本地编辑器，重点放在下面几件事：

- 分镜布局手感：拖拽、缩放、斜切、边中点调整都要直接、稳定。
- 图像贴合准确：画布中看到的分镜形状、裁剪结果、导出结果尽量保持一致。
- 本地项目可靠保存：不依赖外部服务，不把创作流程绑在在线接口上。
- 漫画制作效率：更少的阻力，更快把素材拼成页面。

不再提供的能力：

- 内置后端 server
- 在线生图
- 图像去背景
- 图像超分
- 服务地址 / Authorization 配置

## 当前功能清单

### 分镜与布局

- 新建默认分镜
- 整页网格切割
- 选中单个分镜后二次切割
- 手绘拖拽创建分镜
- 分镜旋转与 5° 吸附
- 分镜四角斜切控制，支持向内/向外拖出梯形或平行四边形
- 分镜四边中点手柄，可单独推动某一条边
- 支持批量设置所有分镜的圆角和边框粗细
- 可选 16 倍数尺寸吸附，方便适配下游图像工作流

### 图像与裁剪

- 仅支持导入本地图片作为分镜素材
- 图片以等比 `cover` 方式贴合分镜，避免拉伸变形
- 支持透明图片直接显示
- 非破坏裁剪：随时重新进入裁剪状态，不会改写原图
- 手动裁剪框会匹配分镜真实形状，而不是简单矩形
- 当分镜尺寸或比例变化时，裁剪参数会尽量自动重算以保持画面中心

### 文字与标注

- 新增文字对象
- 支持矩形、圆角、圆形气泡
- 背景可设为透明或纯白
- 可调边框粗细、边框颜色、字体、字号
- 支持横排与竖排文字
- 文字通过 Konva 直接渲染，导出时不会丢失

### 页面与导出

- 多页面侧栏管理
- 页面新增、删除、上下移动
- PNG 导出当前页
- PDF 按页面顺序导出整套内容
- 导出时会自动隐藏当前选中的蓝色编辑框

### 编辑器能力

- 撤销 / 重做
- 键盘快捷键：`Cmd/Ctrl + Z`、`Cmd/Ctrl + Shift + Z`、`Ctrl + Y`
- 亮色 / 暗色主题切换
- 页面缩略预览
- 本地消息历史与状态提示

## 保存与加载

OpenKoma 优先使用浏览器本地文件能力保存项目。

### 目录模式

如果浏览器支持 `showDirectoryPicker`，项目会保存为类似结构：

```text
project/
  project.json
  history.log
  images/
```

其中：

- `project.json` 保存页面布局、分镜、文字、裁剪等主数据
- `history.log` 保存撤销/重做历史与消息记录
- `images/` 保存项目引用到的本地图片资源

### 单文件模式

如果当前环境无法使用目录权限，仍可导出 / 导入单个：

- `.openkoma.json`

该模式更适合快速备份或跨机器传递，但大项目推荐优先使用目录模式。

### 资源约束

- 当前只保证本地文件、`blob:`、`data:` 资源能被稳定序列化
- 旧项目如果残留外链图片，保存时可能无法完整物化到项目目录

## 快速开始

### 环境要求

- Node.js 18+
- npm 9+

### 安装依赖

```bash
npm install
```

### 启动开发环境

```bash
npm run dev
```

默认地址：`http://localhost:5173`

### 生产构建

```bash
npm run build
```

### 本地预览构建产物

```bash
npm run preview
```

## 适合谁使用

OpenKoma 目前尤其适合：

- 已经有素材，想快速排成漫画页面的创作者
- 需要大量调整分镜形状和裁剪结果的漫画 / 条漫工作流
- 希望项目文件完全保留在本地的个人创作环境
- 想要一个足够轻、但比普通图片拼版工具更懂“分镜几何”的编辑器
